Коннекторы
~~~~~~~~~~
Типичным средством поставки данных в платформу являются коннекторы.

Коннектор - специализированная программа, предназначенная для чтения данных с какого-либо источника и отправки их в платформу.

Предполагается, но не требуется, что один коннектор работает только с одним источником данных.

Общая логика работы коннекторов реализована в базовом модуле для создания коннекторов
`prs-connector-core <https://github.com/mp-co-ru/prs-connector-core>`__ и описана в документации к нему.

Для того, чтобы коннектор мог работать с платформой, в платформе должна быть создана запись,
содержащая настройки коннектора и список привязанных к нему тегов, для которых коннектор должен читать данные
из источника данных.

Связь по MQTT и коды качества тегов
-----------------------------------
Коннекторы подключаются к платформе по протоколу MQTT. Сообщения коннекторов поступают в платформу
через мост RabbitMQ MQTT plugin → обменник AMQP (обменник ``peresvet``).

Платформа отслеживает наличие связи с коннекторами и при определённых событиях записывает во все
теги, привязанные к коннектору, значение ``null`` с соответствующим кодом качества. Это позволяет
различать «нет данных» из-за потери связи, восстановления связи и т.п.

Коды качества
~~~~~~~~~~~~~

* **100** — потеря связи с коннектором. Срабатывает при получении платформой сообщения о потере
  связи (в т.ч. по Last Will and Testament при нештатном отключении коннектора от MQTT-брокера).

* **101** — восстановление связи с коннектором. Записывается при первом сообщении от коннектора
  после подключения (коннектор считается снова онлайн) во все привязанные к нему теги в виде значения
  ``null`` с кодом качества 101.

Last Will and Testament (LWT)
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

Чтобы при нештатном отключении коннектора (обрыв сети, аварийное завершение процесса, таймаут
keepalive) в привязанные теги записывалось качество **100**, коннектор при подключении к MQTT-брокеру
должен настроить Last Will and Testament:

* **Топик (Will topic):** ``prsConnector/connection_lost/<connector_id>``, где ``<connector_id>`` —
  UUID коннектора в платформе (тот же, что используется в топиках ``conn2prs/<connector_id>``
  и ``prs2conn/<connector_id>``).

* **Тело (Will payload):** JSON-объект ``{"id": "<connector_id>"}``.

При срабатывании LWT брокер публикует это сообщение в топик; через привязку обменника ``amq.topic``
к ``peresvet`` (routing key ``prsConnector.connection_lost.*``) сообщение попадает в платформу,
сервис ``connectors_mqtt_app`` обрабатывает его и записывает во все теги данного коннектора
значение ``null`` с кодом качества 100.

Базовый класс коннекторов `prs-connector-core <https://github.com/mp-co-ru/prs-connector-core>`__
уже настраивает LWT при подключении к платформе по MQTT; при разработке коннектора на его основе
дополнительная настройка не требуется.

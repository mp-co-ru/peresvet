## 1. Цель проекта

Повысить прозрачность производственных потерь и обеспечить управляемость эффективности линии на основе методики OEE (Availability, Performance, Quality), используя платформу Peresvet и интерфейс Grafana.

Проект ориентирован на быстрый практический результат: выявление причин потерь, сокращение времени реакции персонала и подготовка основы для масштабирования на другие линии/участки.

## 2. Исходные задачи

1. Организовать сбор производственных данных из доступных источников (PLC/SCADA/MQTT/БД).  
2. Настроить модель технологического объекта и расчеты OEE.  
3. Подготовить рабочие дашборды и мнемосхемы для оперативного персонала и руководителей.  
4. Сформировать единый контур анализа причин простоев и потерь производительности.  
5. Обучить команду заказчика базовой эксплуатации решения.

## 3. Состав работ (Scope пилота)

### Этап 1. Обследование и проектирование
- Уточнение технологического контура пилотной линии;
- Определение источников данных и состава тегов;
- Согласование целевых KPI и критериев приемки.

### Этап 2. Интеграция и модель
- Подключение к согласованным источникам данных;
- Настройка модели объекта в Peresvet;
- Настройка расчетов Availability/Performance/Quality и итогового OEE.

### Этап 3. Визуализация
- Подготовка дашбордов Grafana для ролей (оператор/мастер/руководитель);
- Подключение панели быстрого просмотра тегов (`Теги.json`);
- Настройка мнемосхемы (включая SVG-подход при необходимости).

### Этап 4. Приемка и передача в эксплуатацию
- Функциональные испытания по чек-листу;
- Обучение пользователей;
- Передача документации и итогового отчета по пилоту.

## 4. Результаты проекта (Deliverables)

- Развернутый и настроенный пилотный контур Peresvet для 1 линии;
- Настроенные расчеты OEE и структура причин потерь;
- Рабочие дашборды и мнемосхема для ежедневного использования;
- Комплект эксплуатационной документации;
- Протокол обучения и итоговый акт сдачи-приемки.

## 7. Лицензионная модель при масштабировании

После успешного пилота предлагается переход на годовую подписку:

- **Peresvet Pro** — для одной площадки и ограниченного масштаба;  
- **Peresvet Enterprise** — для отказоустойчивого и масштабируемого контура (HA/кластер, расширенная безопасность, SLA).

Итоговая стоимость определяется по параметрам:
- количество площадок;
- количество производственных линий;
- уровень SLA (8x5 / 24x7);
- дополнительные требования по безопасности и регламентам.

3. Исполнитель не несет ответственность за недостоверность исходных данных заказчика.
